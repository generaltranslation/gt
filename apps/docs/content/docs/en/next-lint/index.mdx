# GT Next Lint

ESLint plugin for General Translation Next.js integration that ensures proper usage of GT Next components.

## Installation

```bash
npm install --save-dev @generaltranslation/gt-next-lint
```

If you don't already have the TypeScript ESLint dependencies:

```bash
npm install --save-dev @typescript-eslint/eslint-plugin @typescript-eslint/parser
```

## Configuration

### Basic ESLint Config

Add to your `eslint.config.mjs`:

```javascript
import gtNext from "@generaltranslation/gt-next-lint";
import tseslint from "@typescript-eslint/eslint-plugin";
import tsparser from "@typescript-eslint/parser";

export default [
  {
    files: ['**/*.{js,jsx,ts,tsx}'],
    plugins: {
      'gt-next': gtNext,
      '@typescript-eslint': tseslint,
    },
    languageOptions: {
      ecmaVersion: 2020,
      sourceType: 'module',
      parser: tsparser,
      parserOptions: {
        ecmaFeatures: {
          jsx: true,
        },
      },
    },
    rules: {
      'gt-next/no-unwrapped-dynamic-content': 'warn',
    },
  },
];
```

### Next.js Template Config

If you're using the standard Next.js ESLint setup with `@eslint/eslintrc`, add GT Next Lint like this:

```javascript
import { dirname } from "path";
import { fileURLToPath } from "url";
import { FlatCompat } from "@eslint/eslintrc";
import gtNext from "@generaltranslation/gt-next-lint";
import tseslint from "@typescript-eslint/eslint-plugin";
import tsparser from "@typescript-eslint/parser";

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

const compat = new FlatCompat({
  baseDirectory: __dirname,
});

const eslintConfig = [
  ...compat.extends("next/core-web-vitals", "next/typescript"),
  {
    plugins: {
      'gt-next': gtNext,
      '@typescript-eslint': tseslint,
    },
    languageOptions: {
      parser: tsparser,
      parserOptions: {
        ecmaFeatures: {
          jsx: true,
        },
      },
    },
    rules: {
      'gt-next/no-unwrapped-dynamic-content': 'warn',
    },
  },
];

export default eslintConfig;
```

## Rule: `no-unwrapped-dynamic-content`

Ensures dynamic content within `<T>` components is wrapped in variable components like `<Var>`, `<DateTime>`, `<Num>`, or `<Currency>`.

### ✅ Correct

```jsx
import { T, Var } from 'gt-next';

<T>Hello <Var>{userName}</Var>!</T>
```

### ❌ Incorrect  

```jsx
import { T } from 'gt-next';

<T>Hello {userName}!</T>  // Warning: wrap {userName} in <Var>
```

### Supported Variable Components

- `<Var>` - General variables
- `<DateTime>` - Date and time values  
- `<Num>` - Numeric values
- `<Currency>` - Currency amounts

Works with imports from `gt-next`, `gt-next/client`, `gt-next/server`, and namespace imports (`import * as GT from 'gt-next'`).
