## Introduction

The `generaltranslation` library serves as GT's core i18n library housing utility functions and classes for translation and formatting.
It is most oftenly used with framework packages like `gt-next` and `gt-react`, but can be used as a standalone library.


import Video from '@/components/Video';

<Video src="/core-demo.mp4" />

```typescript title="index.ts"
import { GT } from 'generaltranslation';

const gt = new GT({
  apiKey: 'your-api-key',
  projectId: 'your-project-id',
  sourceLocale: 'en',
  targetLocale: 'es'
});

// Translate content
const result = await gt.translate('Hello, world!', 'es');
// "¡Hola, mundo!"

// Format numbers, dates, currencies
const formattedPrice = gt.formatCurrency(29.99, 'USD');
const formattedDate = gt.formatDateTime(new Date());
// "$29.99"
// "9/25/2025"

// Work with locales
const localeProps = gt.getLocaleProperties('fr-CA');
const isValid = gt.isValidLocale('de');
// { language: "fr", region: "CA", ... }
// true
```

## Configuration

Configuration options via [GTConstructorParams](/docs/core/types/GTConstructorParams):

| Option | Type | Description |
|--------|------|-------------|
| `apiKey` | `string` | Production API key |
| `devApiKey` | `string` | Development API key |
| `projectId` | `string` | Project identifier |
| `sourceLocale` | `string` | Default source language |
| `targetLocale` | `string` | Default target language |
| `locales` | `string[]` | Supported locales list |
| `baseUrl` | `string` | Custom API endpoint |
| `customMapping` | `CustomMapping` | Custom locale definitions |

## Examples

There are two major types of translation: string translation and file translation.

### Setup

To enable translation, you need to provide a project ID and API key.
Check out the [`constructor`](/docs/core/class/constructor) method for more information.

```typescript
const gt = new GT({
  apiKey: 'your-api-key',
  projectId: 'your-project-id',
  targetLocale: 'es'
});
```

### String Translation
Check out the [`translate()`](/docs/core/class/methods/translation/translate) method for more information.

```typescript
try {
  const result = await gt.translate('Hello, world!');
  console.log(result); // "¡Hola, mundo!"
} catch (error) {
  console.error('Translation failed:', error.message);
}
```


### File Translation

Files are translated as jobs.
You launch a job by uploading a file.
Uploading many files will launch many jobs.

Check out the [`uploadSourceFiles()`](/docs/core/class/methods/translation/uploadSourceFiles)
and [`checkFileTranslations()`](/docs/core/class/methods/translation/checkFileTranslations) methods for more information.

```typescript
// Files to upload
const files = [
  {
    source: {
      fileName: 'src/components/Button.tsx',
      fileFormat: 'TSX',
      locale: 'en',
      content: '...',
    }
  }
];

// Upload source files
await gt.uploadSourceFiles(files);
```
