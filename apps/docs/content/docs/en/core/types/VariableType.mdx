## Overview

`VariableType` is a string literal union type that specifies how variables should be formatted when rendered in translated content. It determines the locale-appropriate formatting behavior for different types of dynamic values.

```typescript
type VariableType = 'v' | 'n' | 'd' | 'c';
```

This type is used within [`Variable`](/docs/core/types/Variable) objects to control how dynamic values are displayed in different locales.

## Type Definition

### Values

| Value | Type | Description | Example Output |
|-------|------|-------------|----------------|
| `'v'` | **Variable** | Default plain text substitution | `"John"`, `"Hello World"` |
| `'n'` | **Number** | Numeric formatting with locale separators | `"1,234"`, `"45.67"`, `"89%"` |
| `'d'` | **Date** | Date/time formatting based on locale | `"March 15, 2024"`, `"15/03/2024"` |
| `'c'` | **Currency** | Monetary value with currency symbols | `"$19.99"`, `"€15.50"`, `"¥1000"` |

## Example

### Basic Variable Type Usage

```typescript copy
import { Variable, VariableType } from 'generaltranslation';

// Plain text variable (default)
const textVariable: Variable = {
  k: 'userName',
  v: 'v'
};

// Number variable for counts and measurements
const numberVariable: Variable = {
  k: 'itemCount',
  v: 'n'
};

// Date variable for timestamps
const dateVariable: Variable = {
  k: 'createdAt', 
  v: 'd'
};

// Currency variable for prices
const currencyVariable: Variable = {
  k: 'totalPrice',
  v: 'c'
};
```

### Formatting Examples by Type

```typescript copy
// Demonstrate different formatting behaviors
const formatExamples = {
  // Variable ('v') - Plain text
  name: { k: 'name', v: 'v' as VariableType },
  // Input: "Sarah"   → Output: "Sarah"
  // Input: "Company Inc." → Output: "Company Inc."
  
  // Number ('n') - Locale-aware numeric formatting  
  quantity: { k: 'qty', v: 'n' as VariableType },
  // Input: 1234      → Output: "1,234" (US), "1.234" (DE)
  // Input: 45.67     → Output: "45.67" (US), "45,67" (FR)
  // Input: 0.15      → Output: "15%" (when used as percentage)
  
  // Date ('d') - Locale-aware date formatting
  eventDate: { k: 'date', v: 'd' as VariableType },
  // Input: Date      → Output: "3/15/2024" (US), "15/03/2024" (UK)
  // Input: Date      → Output: "March 15, 2024" (long format)
  
  // Currency ('c') - Locale-aware currency formatting  
  price: { k: 'cost', v: 'c' as VariableType }
  // Input: 19.99     → Output: "$19.99" (US), "19,99 €" (DE)
  // Input: 1000      → Output: "¥1,000" (JP), "£1,000" (UK)
};
```

### Variable Type Validation

```typescript copy
function validateVariableType(type: any): type is VariableType {
  return typeof type === 'string' && ['v', 'n', 'd', 'c'].includes(type);
}

function createTypedVariable(
  key: string, 
  type: VariableType, 
  id?: number
): Variable {
  if (!validateVariableType(type)) {
    throw new Error(`Invalid variable type: ${type}`);
  }
  
  const variable: Variable = { k: key, v: type };
  
  if (id !== undefined) {
    variable.i = id;
  }
  
  return variable;
}

// Usage examples
const typedVariables = {
  userName: createTypedVariable('userName', 'v'),
  age: createTypedVariable('age', 'n'),
  birthday: createTypedVariable('birthday', 'd'),
  salary: createTypedVariable('salary', 'c')
};
```

## Notes

* Variable types affect formatting behavior but not the underlying data structure
* The `'v'` type is the default and can often be omitted from Variable objects
* Number formatting includes thousands separators, decimal points, and percentage symbols based on locale
* Date formatting automatically respects locale-specific date/time conventions  
* Currency formatting includes appropriate symbols, positioning, and decimal places for each locale
* Variable types work seamlessly with both JSX content and ICU message formats
