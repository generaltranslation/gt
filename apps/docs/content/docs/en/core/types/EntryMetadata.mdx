## Overview

`EntryMetadata` provides optional configuration to customize translation behavior for individual [`Entry`](/docs/core/types/Entry) objects in batch translation operations. It allows fine-grained control over translation models, formats, locales, and caching.

```typescript
type EntryMetadata = {
  context?: string;
  id?: string;
  hash?: string;
  dataFormat?: DataFormat;
  sourceLocale?: string;
  actionType?: ActionType;
  timeout?: number;
  regionCode?: string;
  scriptCode?: string;
};
```

## Type Definition

### Properties

| Property | Type | Description |
|----------|------|-------------|
| `context?` | `string` | Additional context information to help translators understand the content |
| `id?` | `string` | Unique identifier for the translation entry |
| `hash?` | `string` | Content hash for caching and deduplication |
| `dataFormat?` | `DataFormat` | Explicit format specification ('JSX', 'ICU', 'I18NEXT') |
| `sourceLocale?` | `string` | Source locale override for this specific entry |
| `actionType?` | `ActionType` | Translation model preference ('standard', 'fast', or custom) |
| `timeout?` | `number` | Request timeout in milliseconds for this entry |
| `regionCode?` | `string` | ISO region code for locale-specific customization |
| `scriptCode?` | `string` | ISO script code for writing system specification |

### Related Types

#### DataFormat
```typescript
type DataFormat = 'JSX' | 'ICU' | 'I18NEXT';
```

#### ActionType
```typescript
type ActionType = 'standard' | 'fast' | string;
```

- `'standard'` - Higher quality translation model (slower)
- `'fast'` - Faster translation model (lower latency)
- `string` - Custom model identifier

## Example

### Basic Context and ID

```typescript copy
import { Entry, EntryMetadata } from 'generaltranslation';

const metadataExample: EntryMetadata = {
  context: 'Button text in the main navigation menu',
  id: 'nav-home-button'
};

const entry: Entry = {
  source: 'Home',
  targetLocale: 'es',
  metadata: metadataExample
};
```

### Performance Optimization

```typescript copy
const performanceMetadata: EntryMetadata = {
  actionType: 'fast', // Use faster model
  timeout: 5000,      // 5 second timeout
  context: 'Simple UI text that needs quick translation'
};

const quickEntry: Entry = {
  source: 'Loading...',
  targetLocale: 'fr',
  metadata: performanceMetadata
};
```

### Content Format Specification

```typescript copy
const icuMetadata: EntryMetadata = {
  dataFormat: 'ICU',
  context: 'Plural form for item count in shopping cart',
  actionType: 'standard' // Use standard model for complex plurals
};

const pluralEntry: Entry = {
  source: 'You have {itemCount, plural, =0 {no items} =1 {one item} other {{itemCount} items}} in your cart',
  targetLocale: 'de',
  metadata: icuMetadata
};
```

### Timeout Strategy

```typescript copy
function createEntryWithTimeout(
  source: string, 
  targetLocale: string, 
  complexity: 'simple' | 'medium' | 'complex'
): Entry {
  const timeouts = {
    simple: 3000,   // 3 seconds
    medium: 10000,  // 10 seconds  
    complex: 30000  // 30 seconds
  };
  
  const actionTypes = {
    simple: 'fast' as const,
    medium: 'standard' as const,
    complex: 'standard' as const
  };
  
  return {
    source,
    targetLocale,
    metadata: {
      timeout: timeouts[complexity],
      actionType: actionTypes[complexity],
      context: `${complexity} content requiring ${timeouts[complexity]}ms timeout`
    }
  };
}

const timedEntries = [
  createEntryWithTimeout('OK', 'ja', 'simple'),
  createEntryWithTimeout('Please confirm your email address', 'ja', 'medium'),
  createEntryWithTimeout('This legal agreement governs...', 'ja', 'complex')
];
```

## Notes

* All `EntryMetadata` properties are optional - only specify what you need to override
* The `hash` field is useful for implementing translation caching systems
* `actionType` affects both translation quality and processing time
* `regionCode` and `scriptCode` enable locale variants (e.g., 'es-MX' vs 'es-ES')  
* `dataFormat` should match your content type for optimal results
* `timeout` values should account for content complexity and network latency
* `sourceLocale` overrides the GT instance's default source locale for specific entries
