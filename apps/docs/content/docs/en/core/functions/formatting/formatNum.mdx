## Overview

The standalone `formatNum()` function formats numbers according to locale-specific conventions without requiring a GT instance. It provides the same functionality as the GT class method but can be used independently.

```typescript
import { formatNum } from 'generaltranslation';

const formatted = formatNum(1234.56, {
  locales: 'de-DE',
  style: 'currency',
  currency: 'EUR'
});
// Returns: "1.234,56 €"
```

<Callout>
**Standalone Usage:**
This function requires explicit locale specification since it doesn't inherit from a GT instance. It's ideal for utility functions, libraries, or when you need formatting without full GT functionality.
</Callout>

## Reference

### Parameters

| Name | Type | Description |
|------|------|-------------|
| `number` | `number` | The number to format |
| `options` | `NumberFormatOptions & { locales: string \| string[] }` | Formatting configuration with required locales |

### NumberFormatOptions

| Name | Type | Description |
|------|------|-------------|
| `locales` | `string \| string[]` | **Required** - Locales for formatting |
| `style?` | `'decimal' \| 'currency' \| 'percent' \| 'unit'` | Number formatting style |
| `currency?` | `string` | Currency code (required when style is 'currency') |
| `minimumIntegerDigits?` | `number` | Minimum number of integer digits (1-21) |
| `minimumFractionDigits?` | `number` | Minimum number of fraction digits (0-20) |
| `maximumFractionDigits?` | `number` | Maximum number of fraction digits (0-20) |
| `useGrouping?` | `boolean \| 'always' \| 'auto' \| 'min2'` | Whether to use grouping separators |
| `notation?` | `'standard' \| 'scientific' \| 'engineering' \| 'compact'` | Number notation format |

### Returns

`string` - The formatted number according to locale conventions.

## Example

### Basic Usage

```typescript copy
import { formatNum } from 'generaltranslation';

// Basic number formatting
console.log(formatNum(1234.567, { locales: 'en-US' }));
// Output: "1,234.567"

// German formatting
console.log(formatNum(1234.567, { locales: 'de-DE' }));
// Output: "1.234,567"

// Multiple locale fallbacks
console.log(formatNum(1234.567, { locales: ['fr-CA', 'fr-FR', 'en'] }));
// Output: Uses French Canadian if supported, otherwise French, otherwise English
```

### Currency Formatting

```typescript copy
// US Dollar
console.log(formatNum(1234.56, {
  locales: 'en-US',
  style: 'currency',
  currency: 'USD'
}));
// Output: "$1,234.56"

// Euro with German locale
console.log(formatNum(1234.56, {
  locales: 'de-DE',
  style: 'currency',
  currency: 'EUR'
}));
// Output: "1.234,56 €"

// Japanese Yen
console.log(formatNum(1234.56, {
  locales: 'ja-JP',
  style: 'currency',
  currency: 'JPY'
}));
// Output: "¥1,235"
```

### Utility Functions

```typescript copy
// Create locale-specific formatters
function createNumberFormatter(locale: string) {
  return {
    decimal: (num: number) => formatNum(num, { locales: locale }),
    currency: (num: number, currency: string) => formatNum(num, {
      locales: locale,
      style: 'currency',
      currency
    }),
    percent: (num: number) => formatNum(num, {
      locales: locale,
      style: 'percent'
    })
  };
}

const usFormatter = createNumberFormatter('en-US');
const deFormatter = createNumberFormatter('de-DE');

console.log(usFormatter.decimal(1234.56)); // "1,234.56"
console.log(deFormatter.decimal(1234.56)); // "1.234,56"

console.log(usFormatter.currency(1234.56, 'USD')); // "$1,234.56"
console.log(deFormatter.currency(1234.56, 'EUR')); // "1.234,56 €"
```

### Cross-Platform Number Formatting

```typescript copy
// Format numbers for multiple locales simultaneously
function formatForMultipleLocales(number: number, locales: string[]) {
  return locales.map(locale => ({
    locale,
    formatted: formatNum(number, { locales: locale })
  }));
}

const results = formatForMultipleLocales(1234567.89, [
  'en-US', 'de-DE', 'fr-FR', 'ja-JP', 'ar-SA'
]);

results.forEach(({ locale, formatted }) => {
  console.log(`${locale}: ${formatted}`);
});
// Output:
// en-US: 1,234,567.89
// de-DE: 1.234.567,89
// fr-FR: 1 234 567,89
// ja-JP: 1,234,567.89
// ar-SA: ١٬٢٣٤٬٥٦٧٫٨٩
```

## Notes

* Unlike the GT class method, the `locales` parameter is required
* Uses the same underlying `Intl.NumberFormat` as the GT class method
* Results are cached internally for performance with repeated locale/options combinations
* Fallback locales are processed in order if the primary locale is not supported
* All standard `Intl.NumberFormat` options are supported
