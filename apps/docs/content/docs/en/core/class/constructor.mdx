## Overview

The `GT` constructor creates a new instance of the General Translation class, which provides access to all translation, formatting, and locale functionality.

```typescript
import { GT } from 'generaltranslation';

const gt = new GT({
  apiKey: 'your-api-key',
  projectId: 'your-project-id',
  sourceLocale: 'en',
  targetLocale: 'es',
  locales: ['en', 'es', 'fr']
});
```

The constructor automatically reads environment variables for API credentials and validates all provided locale codes.

<Callout>
**Environment Variables:**
The GT constructor automatically reads `GT_API_KEY`, `GT_DEV_API_KEY`, and `GT_PROJECT_ID` from environment variables if not provided in the constructor parameters.
</Callout>

## Behavior

### Environment Variable Loading

The constructor automatically loads configuration from environment variables:

1. **API Key**: Uses `GT_API_KEY` if no `apiKey` parameter provided
2. **Dev API Key**: Uses `GT_DEV_API_KEY` if no `devApiKey` parameter provided  
3. **Project ID**: Uses `GT_PROJECT_ID` if no `projectId` parameter provided

### Locale Validation

All provided locale codes are automatically validated:

- **Source Locale**: Validated against BCP 47 standard and custom mappings
- **Target Locale**: Validated against BCP 47 standard and custom mappings
- **Locales Array**: Each locale is validated, invalid locales cause constructor to throw

### Rendering Locales Setup

The constructor automatically sets up internal rendering locales in this order:
1. Source locale (if provided)
2. Target locale (if provided)  
3. Library default locale (`en`)

## Examples

### Basic Usage

```typescript
import { GT } from 'generaltranslation';

// Minimal setup - uses environment variables
const gt = new GT();
```

### With API Credentials

```typescript

  apiKey: 'gt_prod_abc123...',
  projectId: 'my-project-id',
  sourceLocale: 'en-US',
  targetLocale: 'fr-FR'
```

## Notes

- The constructor validates all locale codes immediately, throwing errors for invalid codes
- Environment variables are only read once during construction
- Custom mappings take precedence over standard BCP 47 validation
- The `baseUrl` parameter is primarily for enterprise customers with custom deployments
- All parameters are optional, but API operations will require `apiKey` and `projectId`
